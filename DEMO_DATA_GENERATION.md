# Demo Data Generation

This document explains how FuelFinder demo data is generated for local SQLite,
local Oracle Docker, and deployment smoke testing.

## Purpose

The demo dataset is synthetic. It is designed to make the application useful for
testing search, station filtering, analytics charts, and forecast logic without
depending on a live fuel-price provider.

The data follows realistic European fuel-price ranges and seasonal behavior, but
it is not a copy of real pump-price history. Station addresses and coordinates
are approximate demo values used to model realistic geographic distribution.

## Dataset Shape

The seed job creates:

- 25 demo stations
- 15 stations in Riga
- 10 stations outside Riga
- 5 fuel station brands: Neste, Circle K, Virsi, Viada, Kool
- 6 fuel types: diesel, diesel_plus, petrol95, petrol98, lpg, electric
- 88 station-fuel links
- 88 current demo price records
- 32,208 historical demo price records

Electric charging is intentionally empty in the first demo dataset. The
`electric` fuel type exists, but no stations are linked to it and no price
records are generated for it. This gives the frontend and backend a stable empty
state to handle.

## Station-Fuel Links

A station-fuel link means that a station offers a specific fuel type.

Example:

```text
Neste Skanste -> diesel
Neste Skanste -> diesel_plus
Neste Skanste -> petrol95
Neste Skanste -> petrol98
```

Not every station offers every fuel type. This makes filtering more realistic:
diesel returns many stations, LPG returns fewer stations, and electric returns
an empty result.

## Current Price Records

Current price records represent the latest visible price for each station-fuel
pair. The search and station endpoints use these records.

There is one current price record for every station-fuel link:

```text
88 station-fuel links = 88 current demo price records
```

The seed source for these records is:

```text
demo_seed_current
```

When the seed job inserts a new current demo record, older current records for
the same station-fuel pair are marked as not current. This keeps manual reruns
safe without dropping tables.

## Historical Price Records

Historical price records are generated for analytics and forecast testing.

The fixed date range is:

```text
2025-06-01 -> 2026-06-01
```

The range is inclusive, so it contains 366 daily records per station-fuel link.

```text
88 station-fuel links * 366 days = 32,208 historical demo price records
```

The seed source for these records is:

```text
demo_seed_history
```

The fixed date range keeps the dataset reproducible. Re-running the seed job on
a later calendar day does not create a new moving history window.

## Price Formula

Historical and current demo prices are generated by the same deterministic
formula:

```text
price =
  base_price_by_fuel
  + brand_modifier
  + station_modifier
  + yearly_trend
  + seasonal_wave_1
  + seasonal_wave_2
  + monthly_event_modifier
  + deterministic_noise
```

The final value is rounded to three decimal places and clamped to a minimum
price per fuel type.

## Base Prices

Base prices are set around realistic Latvia/EU demo ranges:

```text
diesel       = 1.541 EUR/L
diesel_plus  = 1.619 EUR/L
petrol95     = 1.591 EUR/L
petrol98     = 1.688 EUR/L
lpg          = 0.765 EUR/L
```

Electric charging has no generated prices in the current dataset.

## Brand Modifiers

Brand modifiers create small systematic differences between fuel station
brands:

```text
Neste    = -0.010
Circle K = +0.018
Virsi    = -0.006
Viada    = -0.018
Kool     = -0.012
```

These values are synthetic and only help the UI show meaningful price
differences.

## Station Modifier

The station modifier gives each station a small stable offset:

```text
((station_index % 7) - 3) * 0.004
```

This prevents every station from having identical prices under the same brand.

## Yearly Trend

The generated year includes a small gradual upward trend:

```text
yearly_progress * 0.035
```

`yearly_progress` moves from `0.0` at the start of the generated period to `1.0`
at the end.

## Seasonal Waves

Two sine waves create smooth non-linear movement:

```text
sin(day_index / 17.0 + station_index) * 0.012
sin(day_index / 43.0 + station_index * 0.7) * 0.018
```

These waves make graphs less flat and give the forecast algorithm more useful
input than a simple straight line.

## Monthly Events

The generator adds larger monthly events to mimic realistic fuel-price shocks:

```text
2025-07: summer travel bump
2025-09: post-summer drop
2025-11: diesel-specific jump
2026-01: winter/logistics bump
2026-03: upward volatility
2026-04: large Latvia/EU-style fuel jump
2026-05: stabilization with elevated prices
2026-06: continued elevated level
```

These events are synthetic, but their shape is based on realistic European fuel
market behavior and public Latvia/EU price trend references.

## Deterministic Noise

Noise is deterministic, not random. It depends on fuel type, brand, station
index, and day index:

```text
mixed = (seed + station_index * 31 + day_index * 17) % 23
noise = (mixed - 11) * 0.0015
```

This keeps the generated prices stable between seed runs.

## Rebuild Demo Data

Run the seed job from the backend directory:

```powershell
python -m app.jobs.seed_database
```

Then run backend tests:

```powershell
python -m pytest
```

For a quick count check:

```powershell
python -c "from app.database import SessionLocal; from app.models.station_model import Station; from app.models.price_record_model import PriceRecord; from app.models.fuel_type_model import FuelType; db=SessionLocal(); print(db.query(Station).count()); print(db.query(PriceRecord).filter(PriceRecord.source=='demo_seed_history').count()); print(db.query(PriceRecord).join(FuelType).filter(FuelType.code=='electric').count()); db.close()"
```

Expected demo history count:

```text
32208
```
